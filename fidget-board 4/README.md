# 나만의 피젯보드 ASMR 🎧

사운드디자인 수업 과제용으로 만든 웹 피젯보드 뼈대예요.
HTML/CSS를 몰라도 따라올 수 있게 아주 천천히 설명할게요.

---

## 1. 먼저 파일이 뭐가 있는지 보기

```
fidget-board/
├── index.html      ← 페이지의 "뼈대" (뭐가 어디에 있는지)
├── style.css       ← 페이지의 "디자인" (색깔, 모양, 배치)
├── script.js       ← 페이지의 "동작" (클릭하면 뭐가 일어나는지)
├── README.md       ← 지금 읽고 있는 이 설명서
└── assets/
    ├── images/     ← 나중에 넣을 그림/사진 파일
    └── sounds/     ← 나중에 넣을 소리 파일 (효과음 + 배경음악)
```

집을 짓는다고 생각하면 쉬워요.

- **HTML(`index.html`)** = 벽돌과 골조.
- **CSS(`style.css`)** = 페인트, 조명, 가구 배치.
- **JS(`script.js`)** = 전기와 스위치.

세 파일이 서로 연결되어 있어서, 하나만 있으면 안 되고 셋이 같이 있어야 페이지가 제대로 동작해요.

## 2. 지금 화면 구조

상단 바 없이, 화면 가운데에 **보드 + 양옆 스티커**만 있어요.

- **가운데 보드**: 실제로 요소들을 만지작거리는 공간. 왼쪽 위엔 작은 점 4개(보드 디자인 고르기), 오른쪽 위엔 동그라미 버튼 2개(↺ 초기화, 🔈 배경음악)가 있어요.
- **양옆 스티커**: 상자 없이 보드 좌우에 살짝 기울어진 채로 떠 있는 도구들. **옛날 "꾸미기" 플래시게임처럼, 손가락/마우스로 스티커를 눌러서 보드 안까지 직접 끌고 가서 놓아야** 보드에 올라가요. 스티커 자체는 원래 자리에 그대로 남아있어서 같은 종류를 여러 번 가져다 놓을 수 있어요.

## 3. 일단 실행해보기

1. `index.html` 파일을 더블클릭하면 브라우저에서 열려요.
2. 양옆 스티커 하나를 **누른 채로 보드 안까지 끌고 가서 놓아**보세요 → 놓은 그 자리에 요소가 생겨요. (보드 밖에서 손을 떼면 취소돼요)
3. 보드 위의 요소를 **살짝 클릭**하면 → 통통 튀는 애니메이션 + 소리가 나요. 왼쪽에 있으면 왼쪽 스피커, 오른쪽에 있으면 오른쪽 스피커에서 소리가 나니 헤드폰으로 들어보세요. (아직 진짜 소리 파일이 없어서 지금은 임시 "삐" 소리예요)
4. 보드 위 요소를 **눌러서 드래그**하면 원하는 위치로 다시 옮길 수 있어요.
5. 요소에 마우스를 올리면 **×** 버튼이 나타나요 → 누르면 삭제돼요.
6. 보드 오른쪽 위 **↺** 버튼을 누르면 보드를 통째로 비워요(초기화).
7. 보드 오른쪽 위 **🔈** 버튼을 누르면 배경음악이 켜져요(파일 넣었을 때만 소리가 나요).
8. 보드 왼쪽 위 점 4개를 눌러보면 **보드 디자인만** 바뀌어요.
9. 새로고침 해보세요 — 방금 올려둔 요소들이 **그대로 남아있을 거예요.**

> ⚠️ 참고: Claude 채팅 안의 미리보기 화면에서는 브라우저 안전 정책 때문에 "자동 기억하기(localStorage)" 기능이나 소리, 드래그 앤 드롭이 정상 동작하지 않을 수 있어요. `index.html`을 실제로 다운로드해서 컴퓨터에서 직접 열면 전부 정상적으로 동작합니다.

## 4. "웹 자체에서 기억하기" 기능 (자동저장)

요청하신 대로 JSON 파일 다운로드 방식 대신, 브라우저의 **localStorage**라는 저장 공간에 보드 상태를 자동으로 저장하도록 만들었어요.

- 요소를 추가/이동/삭제할 때마다 자동으로 저장됨
- 페이지를 새로고침하거나 나중에 다시 열어도 그대로 복원됨
- 별도의 저장/불러오기 버튼 없이 그냥 알아서 기억함

**딱 하나 알아둘 점**: `localStorage`는 "그 컴퓨터의 그 브라우저"에만 저장돼요. 다른 컴퓨터나 다른 브라우저에서 열면 빈 보드로 시작해요. (선생님께 파일로 제출할 때는 이 점은 문제되지 않아요 — 선생님이 그 파일을 여신 컴퓨터 기준으로 새로 시작할 뿐이에요.)

만약 "친구한테 내 보드를 파일로 보내주고 싶다" 같은 기능이 나중에 필요해지면, 예전 방식(JSON 다운로드)을 다시 추가할 수도 있어요. 지금은 요청하신 대로 심플하게 자동저장 하나만 남겨뒀어요.

## 5. 보드 디자인(테마) 바꾸기

보드 왼쪽 위의 작은 점 4개는 **가운데 보드의 배경**만 바꿔요.

- 🔵 스노우 (기본, 레퍼런스 이미지 같은 하늘색+점무늬)
- 🟣 한밤중
- 🟤 우드보드
- 🩷 네온

**동작 원리**: 버튼을 누르면 `<body>`의 `data-theme` 값이 바뀌고, `style.css`의 `body[data-theme="..."] { ... }` 블록이 보드 배경 색상 변수들을 덮어써요.

**새 테마 추가하기**: `style.css`에서 테마 블록 하나(예: `body[data-theme="wood"] { ... }`)를 복사해서 이름과 색상 값만 바꾸고, `index.html`의 `theme-switcher` 안에 버튼을 하나 추가하면 돼요.

```html
<button class="theme-dot" data-theme-choice="candy" style="background:#ff5b6e" type="button" title="캔디"></button>
```

보드 배경 자체를 그림(이미지)으로 바꾸고 싶다면, `style.css`의 `.board`에서 `background-image` 부분을 원하는 이미지 경로로 바꾸면 돼요.

```css
.board {
  background-image: url('assets/images/my-board-bg.png');
  background-size: cover;
}
```

## 6. 좌우 스피커(스테레오 사운드) — 이미 들어가 있어요 ✅

요소가 보드 왼쪽에 있으면 왼쪽 스피커, 오른쪽에 있으면 오른쪽 스피커에서 더 크게 들리도록 만들어뒀어요. `script.js`의 `getPanForElement()` 함수가 요소의 좌우 위치를 -1(완전 왼쪽) ~ 1(완전 오른쪽) 사이 숫자로 계산하고, `StereoPannerNode`라는 Web Audio API 기능으로 그 숫자를 소리에 적용해요.

> ⚠️ 실제 mp3 파일을 넣었는데 소리/좌우패닝이 안 된다면 `file://`로 직접 연 것 때문일 수 있어요. VSCode의 Live Server 확장이나 깃허브 페이지처럼 실제 서버에 올려서 열면 확실하게 동작해요.

## 7. 나만의 그래픽/사운드로 바꾸기

### 7-1. 효과음 / 배경음악 파일 넣기

1. mp3 파일들을 `assets/sounds/` 폴더에 넣으세요.
2. 파일 이름을 맞춰주세요: `bubble.mp3`, `spinner.mp3`, `switch.mp3`, `squish.mp3`, 그리고 배경음악은 `bgm.mp3`.
3. 끝! 파일만 넣으면 자동으로 연결돼요.

### 7-2. 이모지 대신 직접 그린 그림/사진 넣기

지금은 이모지(🫧, 🌀 등)로 그래픽을 대신하고 있어요. 이미지로 바꾸려면:

1. 이미지 파일을 `assets/images/` 폴더에 넣으세요. 예: `bubble.png`
2. `script.js`의 `addItemToBoard` 함수 안에서 이 부분을 찾아서

   ```js
   el.innerHTML = `
     <span class="item-emoji">${type.emoji}</span>
     <button class="delete-btn" type="button" title="삭제">×</button>
   `;
   ```

   아래처럼 `<img>` 태그로 바꿔주세요.

   ```js
   el.innerHTML = `
     <img class="item-image" src="assets/images/${type.id}.png" alt="${type.label}" />
     <button class="delete-btn" type="button" title="삭제">×</button>
   `;
   ```

3. `style.css` 맨 아래에 추가:

   ```css
   .item-image {
     width: 70%;
     height: 70%;
     object-fit: contain;
     pointer-events: none;
   }
   ```

양옆 스티커(`renderStickers`, `enableStickerDrag` 함수, `.sticker-item`)에도 같은 방법을 적용할 수 있어요.

> 💡 스티커를 끌어다 놓는 동작은 `script.js`의 `enableStickerDrag()` 함수 안에 있어요. 드래그 도중 손가락/마우스를 따라다니는 동그라미는 `.drag-ghost`라는 이름의 임시 요소예요.

### 7-3. 새로운 피젯 요소 종류 추가하기

`script.js` 맨 위 `ELEMENT_TYPES` 배열에 한 줄만 추가하면 새 요소가 생겨요. (짝수 번째는 왼쪽 스티커 줄, 홀수 번째는 오른쪽 스티커 줄에 자동으로 배치돼요)

```js
{ id: 'dial', label: '다이얼', emoji: '🎛️', color: '#c9b8f7', soundFreq: 420, soundFile: 'dial.mp3' },
```

### 7-4. 크기 조절하기 (아주 자세히)

크기는 전부 `style.css`에 있는 숫자(px) 값 하나씩만 바꾸면 돼요. 어디를 바꾸면 뭐가 바뀌는지 표로 정리했어요.

| 바꾸고 싶은 것 | 파일 · 위치 | 속성 | 지금 값 |
|---|---|---|---|
| 보드에 이미 놓인 아이콘 크기 | `style.css` → `.fidget-item` | `width`, `height` | 84px |
| 보드에 놓인 아이콘 안의 이모지 크기 | `style.css` → `.item-emoji` | `font-size` | 34px |
| 양옆에 떠 있는 스티커 크기 | `style.css` → `.sticker-item` | `width`, `height`, `font-size` | 64px / 40px |
| 드래그 중 손가락을 따라다니는 아이콘 크기 | `style.css` → `.drag-ghost` | `width`, `height`, `font-size` | 84px / 46px |
| 보드 전체 가로 최대 크기 | `style.css` → `.board-frame` | `max-width` | 640px |
| 보드 전체 세로 최소 크기 | `style.css` → `.board` | `min-height` | 520px |
| 양옆 스티커 줄의 가로 폭 | `style.css` → `.sticker-rail` | `flex: 0 0 110px;`의 110px | 110px |
| 스티커끼리의 세로 간격 | `style.css` → `.sticker-rail` | `gap` | 22px |

**바꾸는 방법**: `style.css` 파일을 열고 (메모장이나 VSCode 같은 걸로), `Ctrl+F`(찾기)로 위 표의 "속성/클래스 이름"을 검색해서 숫자만 원하는 값으로 고치고 저장하면 끝이에요. 예를 들어 보드 위 아이콘을 더 크게 하고 싶다면:

```css
.fidget-item {
  width: 110px;   /* 84px → 110px 로 변경 */
  height: 110px;  /* 84px → 110px 로 변경 */
}
.item-emoji {
  font-size: 44px; /* 34px → 44px 로 같이 키워줘야 이모지가 원 안에 꽉 차 보여요 */
}
```

> 💡 지금은 일부러 **스티커(64px)는 살짝 작게, 보드에 놓인 아이콘/드래그 중인 아이콘(84px)은 크게** 만들어뒀어요. 마치 작은 스티커를 떼서 큰 자리에 붙이는 느낌을 주려고요. 크기를 전부 통일하고 싶다면 세 군데(`.sticker-item`, `.fidget-item`, `.drag-ghost`) 숫자를 같은 값으로 맞추면 돼요.

## 8. 누르면 찌그러졌다가 통통 튀며 돌아오는 효과 (+ 플랫 디자인)

요청하신 두 가지 다 반영했어요.

**① 누르면 살짝 찌그러지는 효과**: 아이콘(양옆 스티커든, 보드 위에 놓인 요소든)을 마우스/손가락으로 누르고 있는 동안은 살짝 옆으로 퍼지듯 눌린 모양이 됐다가, 손을 떼는 순간 통통 튀면서 원래 모양으로 돌아와요. 이건 두 단계로 구현되어 있어요.

- **누르는 동안**: `.pressed` 라는 클래스가 붙어서 `transform: scale(1.15, 0.82)` 처럼 가로로 넓고 세로로 낮은 모양이 돼요. (`style.css`에서 `.sticker-item.pressed`, `.fidget-item.pressed` 검색)
- **손을 뗀 순간**: `.pressed`가 사라지고 `.pop`이 붙으면서 `pop-rotated`라는 애니메이션이 재생돼요. 커졌다 작아졌다 하면서 통통 튀는 느낌을 만들어요. (`style.css`에서 `@keyframes pop-rotated` 검색)

찌그러지는 정도를 더 세게/약하게 하고 싶으면 `.pressed`의 `scale(1.15, 0.82)` 숫자를 바꿔보세요. 두 번째 숫자(0.82)가 작을수록 더 납작하게 눌려요.

**② 그림자 / 아이콘 테두리 / 둥근 모서리 전부 제거**: 요청하신 대로 보드, 스티커, 보드 위 아이콘, 초기화/배경음악 버튼, 테마 점까지 전부 `box-shadow: none`, `border: none`, `border-radius: 0`으로 바꿨어요. 이제 아이콘 자리에는 배경 원(circle) 없이 이모지(또는 나중에 넣을 이미지) 자체만 둥둥 떠 있어요 — 보내주신 레퍼런스처럼 그림만 오려붙인 스티커 느낌이에요.

혹시 나중에 그림자/테두리가 조금 다시 필요해지면 (예: 아이콘이 배경과 구분이 잘 안 될 때), 이렇게 한 줄만 추가하면 돼요.

```css
.fidget-item {
  filter: drop-shadow(0 4px 6px rgba(0, 0, 0, 0.25)); /* 이모지/이미지 모양 그대로 그림자만 살짝 */
}
```

`box-shadow`는 사각형/원 모양의 그림자를 만들지만, `filter: drop-shadow(...)`는 투명한 부분을 무시하고 **그림/이모지의 실제 모양**을 따라 그림자를 그려줘서 지금처럼 배경이 없는 아이콘에 더 잘 어울려요.

## 10. 추가로 넣으면 재밌을 아이디어

- **뽁뽁이 전용 그리드**: `bubble` 타입만 작은 원 여러 개를 격자로 배치해서 하나씩 개별적으로 "펑" 터뜨릴 수 있게 만들기.
- **볼륨 슬라이더**: 효과음/배경음악 크기를 따로 조절하는 슬라이더.
- **회전 기능**: 스피너류 요소는 드래그로 빙글빙글 돌릴 수 있게.
- **스티커 흔들림 효과**: 마우스를 올리면 스티커가 살짝 흔들리는 애니메이션 추가.
- **직접 녹음하기**: 마이크로 나만의 사운드를 녹음해서 요소에 연결 (`MediaRecorder` API 검색 참고).
- **앞뒤(위아래) 입체감**: 좌우 패닝처럼, 위쪽 요소는 소리가 더 멀고 작게, 아래쪽은 더 가깝고 크게 들리도록 확장.
- **키보드 접근성 강화**: Tab으로 선택 후 화살표 키로 이동까지 가능하게.
